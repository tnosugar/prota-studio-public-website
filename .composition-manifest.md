The 2026-05-12 L9 entry below is preserved for history but **superseded** by this entry — the L9 work codified a spec-aspirational pattern that hadn't been verified against deployed reality; the corrective f40cbe7 pulled the deployed reality up into the library and the present re-emission re-derives the deliverables from the corrected spec.

### What the corrected library changed (from what the L9 entry produced)

- **3-state comment lifecycle.** `pending` / `applied` / `archived` enum, where `applied` is a first-class state (build team responded affirmatively) distinct from `archived` (dismissed). Pre-correction prota was 2-state (boolean `archived`); this re-emission adds the third state plus the matching UI + write paths.
- **Class-prefix scope rule clarified.** `.review-` prefix is on chrome ROOTS only; project-local structural classes inside chrome roots (`.text`, `.actions`, `.anchor`, `.group-status`, etc.) do NOT carry the prefix. The deployed prota + glinda pattern already used unprefixed inner-structure classes; the L9 entry's text-on-paper would have prefixed them but the actual L9 emission never did. Net effect on this re-emission: zero change to selector names.
- **Has-comment outline is class-based, not data-attribute-based.** `.has-comment` / `.has-applied-comment` classes on anchored elements (not `[data-has-comments]` attribute). Already aligned in the deployed files; the L9 text-on-paper would have suggested attributes but the L9 emission never used them.
- **Per-action suffix classes for in-row buttons.** `.apply-btn` / `.edit-btn` / `.archive-btn` / `.restore-btn` / `.delete-btn` for the per-row action buttons (modal buttons keep `.review-btn--primary/--secondary/--danger`). Pre-correction prota had three of the five (edit / restore / delete); this re-emission adds the missing two (apply, archive) for the new pending → applied transition.

## 2026-05-12 re-emission — css-isolation contract (L9) — SUPERSEDED by 2026-05-13

Re-emitted `review-mode.css` and class references in `review-mode.js` against the new `library/features/review-widget/css-isolation.md` atomic (upstream commit `1fa1501`). The L9 spec landed after glinda's third-adoption pass surfaced widget-chrome inconsistency across projects; prota's pre-L9 review-mode.css emitted bare `.review-*` selectors (host-page styles could cascade into widget chrome) and used `opacity`-driven pill hide/show (host-page rules could leave the pill visible after cursor exit). This re-emission honors the contract:

- Every CSS rule is now scoped under `html[data-review-mode="on"]` (108 scope-prefix occurrences in the rewritten file; 0 bare `.review-*` selectors). The bootstrap-set attribute on `<html>` (already present in pre-L9 review-mode.js at line 168) gates the entire stylesheet, so it remains inert until the widget is activated.
- Chrome reset block emitted at top of CSS — `all: revert; font-family: var(--review-font-sans); box-sizing: border-box; line-height: 1.4;` on banner / sidebar / sidebar-toggle / modal / modal-backdrop / toast / pill / pill-container. `box-sizing: border-box` propagated to all descendants. Host-page cascade can no longer bleed into widget chrome.
- Pill baseline changed from `opacity: 0; pointer-events: none;` to `display: none;` (the non-negotiable spec-mandated baseline per `pill-hover-deepest.md` §"Required emission"). Hover override uses `display: inline-flex` via the high-specificity `[data-review-mode="on"] [data-comment-id]:hover:not(:has([data-comment-id]:hover)) > .review-pill-container > .review-pill` selector.
- `.review-pill-container` wrapper introduced. The container is positioned absolutely as a host slot with `pointer-events: none` (empty container doesn't intercept clicks on host element); the pill inside takes `pointer-events: auto`. JS: pill DOM construction in `wireAnchors()` now creates the container first, then appends the pill into it.
- Button isolation block — `.review-btn` base + BEM variants `.review-btn--primary / --secondary / --danger` with explicit `font-family: inherit; background: transparent; color: inherit; border; padding; cursor`. Modal cancel/submit buttons rewritten in JS to emit `class="review-btn review-btn--secondary"` and `class="review-btn review-btn--primary"` instead of `class="secondary"` / `class="primary"`.
- Always-on dashed outline on every `[data-comment-id]`: `outline: 1px dashed var(--review-accent-warm-low); outline-offset: 2px; position: relative;` per `anchor-strategy.md` §"Visual treatment". Pre-L9 prota only emitted outlines on hover and on the `.has-comment` state class; spec requires unconditional dashed outline so any anchored element is visually identifiable at rest. Existing `.has-comment` / `.has-applied-comment` state classes preserved.
- Spotlight class renamed `.review-spotlight` → `.review-spotlit` per `css-isolation.md` §"Spotlight class" (CSS + 4 JS `classList` calls in `spotlightAnchor()`).
- `@media (prefers-reduced-motion: reduce)` guard added for spotlight animation (was missing pre-L9).
- SKIP_SELECTOR in JS extended to include `.review-pill-container` alongside `.review-pill` so the container doesn't self-anchor.
- Brand tokens preserved: prota navy primary-deep `#1F3864`, primary-darker `#142646`, amber `#f59e0b`, sage `#4ade80`, slate neutrals. All values now expressed as `--review-*` custom properties at the top of the file rather than literal hex throughout.

No `review-bootstrap.js` change. The bootstrap's `injectKomentariButton` and conditional CSS/JS load remain intact; the widget's `<html data-review-mode="on">` attribute set lives in `review-mode.js` at the existing line 168.
